## 핵심 UX 개선 포인트

### 1) 스텝퍼 or 섹션 접기
**현재 PC**: 입력 카드 2개(소득 + 공제) 가로 2컬럼 → 모바일에서 세로로 스택되면 스크롤이 매우 길어짐

**모바일 개선안 (택 1)**:
- **A) 스텝퍼**: "1/3 소득 입력 → 2/3 공제 → 3/4 결과" 식으로 단계 분리
- **B) 섹션 접기**: 모든 카드가 기본 접힘, 하나씩 열어서 입력 (자비스앤빌런즈 스타일)
- **C) 진행 상태바**: 상단에 "입력 완료 ●●○○○ 60%" 진행도 표시

추천: **B) 섹션 접기** — 스텝 전환 없이 한 페이지에서 스캔 가능

### 2) 결과 요약 sticky
- 입력 중에도 현재까지 계산된 "예상 세액"이 **하단(또는 상단) sticky**로 실시간 업데이트
- 탭하면 결과 상세 섹션으로 스크롤
- 삼쩜삼의 환급 예상액 상단 고정 UX 벤치마크

### 3) 숫자 입력 UX
- 모든 금액 입력은 `inputMode="numeric"` 유지 (이미 적용됨)
- **자동 천단위 쉼표 표시** (예: 5,000만원)
- 증감 버튼(+/-)은 모바일에 적합, 유지
- 토글 스위치 터치 영역 48px+ 확보

### 4) 결과 흐름 시각화
**현재 PC**: 가로 플로우 "수입 → 경비 → 소득" 화살표로 연결
**모바일**: 세로 타임라인으로 변환 (↓ 아이콘 + 라인)
- 각 단계 금액을 큰 글자로, 차감은 빨간색, 최종은 파란색 강조

### 5) 결과 카드 (세액 3~4개)
- 모바일은 세로 스택, 총 납부세액은 가장 크고 그라데이션 배경
- "지방소득세는 소득세의 10%" 같은 보조 설명은 작은 칩으로

## 벤치마크

- **삼쩜삼** — 세무 계산 UX 전반, 상단 예상환급 고정
- **토스 송금/신청 플로우** — 단계별 입력 UX
- **자비스앤빌런즈 세무 계산기** — 섹션 접기 패턴
- **네이버 금융 계산기** — 심플한 입력 + 즉시 결과
